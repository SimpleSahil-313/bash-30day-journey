# 30 Days of Bash

This repo documents my 30-day challenge to improve Bash scripting by building real scripts that automate daily tasks.

using tutorials only for understanding(because i don't wanna lie by writting no tutorials). No copy-paste. Just writing scripts and learning by doing.

---

## Goal

- Get comfortable writing Bash without relying on constant Googling  
- Build scripts that solve real problems  
- Develop an automation-first mindset  
- Apply scripting to cybersecurity (SOC, Red Team, system tasks)

---

## What I'm Building

- File and log management scripts  
- Backup automation  
- System monitoring tools  
- Basic security-focused scripts  
- Cron-based automation  

---

## Why This Matters

Bash is used everywhere:

- Blue Team → log analysis, monitoring  
- Red Team → enumeration, payload execution  
- DevOps → automation, deployments  

If you can't automate, you're wasting time.

---

## Notes

This is not about writing perfect scripts.

It’s about getting better by building consistently.
